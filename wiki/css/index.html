# CSS Selector priority (!important)

  1. . selects class, # selects id.
  2. selectorA selectorB selects every selectorB component under selectorA.
  3. selectorA > selectorB selects selectorB directly under selectorA
  4. !important has the utmost priority. But it interacts with how CSS works. Should not use unless you must.
    • {property1:value1 !important}
  5. Priority : inline style > idselector > class selector > element selector > writtenorder

# Units of CSS

  1. px : Pixel
  2. % : Percentage of changing layout
  3. em : 'parent element's font-size'
  4. rem : 'root element's font-size'. Preferred over em.
  5. viewport : Visible area of webpage on display device. Data is accessible via HTML meta tag or CSS parameters.
  6. vh, vw : Relative to 1% height/width of the viewport.
  7. vmin, vmax : Relative to 1% of viewport's min/max dimension.

# Box model

  1. Content
  2. Padding
  3. Border
  4. Margin Margin Collapse : When bottom margin of upper box and top margin of lower box meets, they are overlapped(collapsed), resulting in one bigger margin.

# Display

  1. block : Element takes up the whole width like <p> and takes up space like 'blocks'.
  2. inline : Height and width properties have no effect like <span>. Width is decided by content. Because margin-top and margin-bottom doesn't work, you can use line-height to give spacings.
  3. inline-block : Inline, but you can apply height and width values
  4. none : Makes the element disappear. Unlike visibility:hidden, empty space will disappear as well.

# Position

  1. static : default
  2. relative : moves the element relative position from its static(default) position. Default position is occupied with empty space.
  3. absolute : moves the element relative to its parent or <body> element. Default position is filled with other elements. absolute is used for UIs, popups, menus.
  4. fixed : places the element relative to browsers viewport.
